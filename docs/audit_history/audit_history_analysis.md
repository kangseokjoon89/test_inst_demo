# 감사 이력 업무 분석 문서

화면: `감사로그조회`
API: `/audit/history`
테이블: `TBAUDITHIST`
핵심 컬럼: `ACTIONUSERNM`

## 운영 문의 예시

비개발자가 `작업자명 변경 후 과거 감사 로그에는 당시 이름이 남는 상황`라고 문의했을 때 AI는 다음 순서로 근거를 찾는다.

## 분석 원칙

1. 질문에서 기관코드, 기준일, 상태값, 화면명을 먼저 추출한다.
2. Nexacro transaction id를 찾아 호출 URL을 확인한다.
3. Controller와 Service를 거쳐 Mapper XML의 SQL id까지 이동한다.
4. Tibero 기준 JOIN SQL로 현재 기준정보와 snapshot 값을 비교한다.
5. SVN revision 또는 Git commit 기록이 있으면 변경 시점을 함께 본다.
6. 서브쿼리 없이도 확인 가능한 SQL을 먼저 제시한다.
7. snapshot 설계와 실제 오류 가능성을 분리해서 답한다.

## 분석 장면 1: URL 단서

transaction의 두 번째 인자가 백엔드 Controller route와 연결된다.
이 모듈에서는 `TBAUDITHIST.ACTIONUSERNM`가 주된 관찰 대상이다.
사용자가 말한 `운영자 작업 기록` 이후 데이터가 달라졌는지 확인한다.
'운영 문의에는 대개 기관코드, 기준일, 화면명, 담당자명이 함께 포함된다.'
운영자에게는 바로 실행 가능한 SELECT를 제공하고, 개발자에게는 소스 경로를 제공한다.
