# AI 인덱싱 설계 가이드

이 문서는 AI 개발자가 GitHub 또는 SVN 저장소를 수집할 때 어떤 단위로 나누고 어떤 메타데이터를 붙여야 하는지 설명합니다.

## Chunk Rule 1: Java Service

- 권장 단위: 업무 분기 단위
- 필수 메타데이터: snapshot 갱신 여부, 상태 전이, 원인 추론 문장
- 검색 시에는 벡터 검색과 키워드 검색을 같이 사용합니다.
- 테이블명이나 컬럼명이 포함된 질문은 BM25 또는 LIKE 검색 신호를 강하게 반영합니다.
- 업무 의도를 묻는 질문은 Service와 업무문서의 설명을 함께 넣어 답변합니다.

## Chunk Rule 2: Java Mapper

- 권장 단위: 인터페이스 메서드 단위
- 필수 메타데이터: SQL id, 테이블명, 컬럼명
- 검색 시에는 벡터 검색과 키워드 검색을 같이 사용합니다.
- 테이블명이나 컬럼명이 포함된 질문은 BM25 또는 LIKE 검색 신호를 강하게 반영합니다.

## Chunk Rule 3: MyBatis XML

- 검색 시에는 벡터 검색과 키워드 검색을 같이 사용합니다.
